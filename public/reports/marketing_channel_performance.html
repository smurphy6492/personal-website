Methodology

Data Sources: campaigns, pageviews, sessions, transactions

Analysis Approach: To answer the conversion rate by channel and cost-per-acquisition questions, we need two core analyses: (1) session-to-transaction conversion rates grouped by channel, joining sessions and transactions; (2) cost-per-acquisition by channel, joining campaigns spend data with session/transaction counts. Since campaign_id has 57.7% nulls in sessions, CPA analysis will be limited to sessions with attributed campaigns, which we will flag. We will also include a supporting breakdown to cross-validate channel-level spend totals from campaigns.

SQL Queries Used

Calculate the session-to-transaction conversion rate for each channel by counting total sessions and sessions that resulted in a transaction, then dividing to get conversion rate.

SELECT
    s.channel,
    COUNT(DISTINCT s.session_id) AS total_sessions,
    COUNT(DISTINCT t.transaction_id) AS sessions_with_transaction,
    ROUND(
        COUNT(DISTINCT t.transaction_id) * 100.0 / COUNT(DISTINCT s.session_id),
        2
    ) AS conversion_rate_pct
FROM sessions s
LEFT JOIN transactions t
    ON s.session_id = t.session_id
GROUP BY s.channel
ORDER BY conversion_rate_pct DESC

Calculate cost-per-acquisition (CPA) by channel by summing total campaign spend per channel and dividing by the number of converting sessions attributed to those campaigns. Accounts for the 57.7% null campaign_id in sessions by scoping to attributed sessions only.

WITH attributed_sessions AS (
    SELECT
        s.session_id,
        s.campaign_id,
        c.channel,
        c.total_spend
    FROM sessions s
    INNER JOIN campaigns c ON s.campaign_id = c.campaign_id
),
channel_spend AS (
    SELECT
        channel,
        SUM(total_spend) / COUNT(DISTINCT campaign_id) AS channel_spend
    FROM (
        SELECT DISTINCT campaign_id, channel, total_spend
        FROM attributed_sessions
    ) camp_deduped
    GROUP BY channel
),
channel_conversions AS (
    SELECT
        a.channel,
        COUNT(DISTINCT t.transaction_id) AS converting_sessions
    FROM attributed_sessions a
    INNER JOIN transactions t ON a.session_id = t.session_id
    GROUP BY a.channel
),
channel_campaign_spend AS (
    SELECT
        c.channel,
        SUM(c.total_spend) AS total_channel_spend
    FROM campaigns c
    GROUP BY c.channel
)
SELECT
    cv.channel,
    cs.total_channel_spend,
    cv.converting_sessions,
    ROUND(cs.total_channel_spend::DOUBLE / cv.converting_sessions, 2) AS cpa
FROM channel_conversions cv
INNER JOIN channel_campaign_spend cs ON cv.channel = cs.channel
ORDER BY cpa DESC

Summarize total budgeted spend, number of campaigns, and average daily budget per channel from the campaigns table to provide context for CPA calculations and identify which channels have the most investment.

SELECT
    channel,
    COUNT(campaign_id) AS num_campaigns,
    SUM(total_spend) AS total_budgeted_spend,
    AVG(daily_budget) AS avg_daily_budget,
    SUM(days_active) AS total_days_active
FROM campaigns
GROUP BY channel
ORDER BY total_budgeted_spend DESC

Show monthly conversion rate trends by channel to identify whether certain channels improve or degrade over time, providing deeper context beyond the overall conversion rate.

SELECT
    DATE_TRUNC('month', s.session_date) AS month,
    s.channel,
    COUNT(DISTINCT s.session_id) AS total_sessions,
    COUNT(DISTINCT t.transaction_id) AS total_conversions,
    ROUND(
        100.0 * COUNT(DISTINCT t.transaction_id) / NULLIF(COUNT(DISTINCT s.session_id), 0),
        2
    ) AS conversion_rate_pct
FROM sessions s
LEFT JOIN transactions t
    ON s.session_id = t.session_id
GROUP BY
    DATE_TRUNC('month', s.session_date),
    s.channel
ORDER BY
    month ASC,
    s.channel ASC
