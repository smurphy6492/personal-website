Methodology

Data Sources: campaigns, pageviews, sessions, transactions

Analysis Approach: To compare mobile vs. desktop conversion by traffic source, we need to join sessions with transactions to compute conversion rates (sessions that resulted in a purchase) segmented by device type and channel. We will also look at revenue per converting session to understand value differences. A supplementary query will examine session engagement metrics (bounce rate, pages viewed) by device and channel to provide context for conversion differences.

SQL Queries Used

Calculate the conversion rate (sessions with at least one transaction / total sessions) broken down by device type (mobile vs. desktop) and traffic channel, to directly answer how conversion differs across these dimensions.

SELECT
  s.device,
  s.channel,
  COUNT(DISTINCT s.session_id) AS total_sessions,
  COUNT(DISTINCT t.session_id) AS converted_sessions,
  ROUND(100.0 * COUNT(DISTINCT t.session_id) / COUNT(DISTINCT s.session_id), 2) AS conversion_rate_pct
FROM sessions s
LEFT JOIN transactions t
  ON s.session_id = t.session_id
GROUP BY s.device, s.channel
ORDER BY conversion_rate_pct DESC

Calculate average revenue per converting session and average order value segmented by device type and traffic channel, to understand not just whether users convert but how much value they generate when they do.

SELECT
    s.device,
    s.channel,
    COUNT(DISTINCT t.session_id) AS converting_sessions,
    COUNT(t.transaction_id) AS total_orders,
    SUM(t.revenue) AS total_revenue,
    SUM(t.revenue) / COUNT(DISTINCT t.session_id) AS avg_revenue_per_converting_session,
    SUM(t.revenue) / COUNT(t.transaction_id) AS avg_order_value
FROM sessions s
INNER JOIN transactions t ON s.session_id = t.session_id
GROUP BY s.device, s.channel
ORDER BY avg_revenue_per_converting_session DESC

Examine session engagement indicators (bounce rate, average pages viewed, average session duration) by device and channel to provide context for why conversion rates may differ — e.g. higher bounce on mobile may explain lower conversion.

SELECT
    device,
    channel,
    COUNT(session_id) AS total_sessions,
    ROUND(100.0 * SUM(bounced) / COUNT(session_id), 2) AS bounce_rate_pct,
    ROUND(AVG(pages_viewed), 2) AS avg_pages_viewed,
    ROUND(AVG(duration_seconds), 2) AS avg_session_duration_seconds
FROM sessions
GROUP BY device, channel
ORDER BY bounce_rate_pct DESC

Show monthly conversion rate trends split by device type across all channels combined, to identify whether the mobile vs. desktop gap is widening, narrowing, or stable over time.

SELECT
    DATE_TRUNC('month', s.session_date) AS month,
    s.device,
    COUNT(DISTINCT s.session_id) AS total_sessions,
    COUNT(DISTINCT t.transaction_id) AS total_conversions,
    ROUND(100.0 * COUNT(DISTINCT t.transaction_id) / NULLIF(COUNT(DISTINCT s.session_id), 0), 2) AS conversion_rate_pct
FROM sessions s
LEFT JOIN transactions t
    ON s.session_id = t.session_id
GROUP BY
    DATE_TRUNC('month', s.session_date),
    s.device
ORDER BY
    month ASC,
    s.device ASC
